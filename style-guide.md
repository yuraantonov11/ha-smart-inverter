# 🎨 Smart Inverter Desktop — Style Guide

---

## 1. Загальна філософія дизайну

**Назва стилю:** Clean Tech / Cyberpunk-Light Dashboard  
**Настрій:** Професійний командний центр керування енергією — футуристичний, але не агресивний. Відчуття "преміального технічного інструменту", а не "ігрової панелі".  
**Ключові принципи:**
- **Glassmorphism** — напівпрозорі поверхні з розмиттям фону
- **Semantic Neon** — кожен тип енергії має свій колір-акцент із неоновим свіченням
- **Data-First** — дані завжди на першому плані, декор підтримує, а не відволікає
- **Controlled Darkness** — темний фон створює контраст для яскравих акцентів

## 3. Типографіка

### 3.1 Display Font — Orbitron

- **Призначення:** Заголовки, мітки навігації, назви енергетичних вузлів, технічні лейбли
- **Стиль:** Геометричний, квадратний, футуристичний
- **Ваги:** 400–900
- **Характеристики використання:**
  - Завжди `UPPERCASE` або `tracking-wider` (збільшений міжлітерний інтервал)
  - Розміри: `text-[9px]` для мікролейблів, `text-xs` для міток вузлів, `text-sm`–`text-lg` для заголовків секцій
  - Ніколи не використовувати для довгих текстів або абзаців

### 3.2 Body Font — Inter

- **Призначення:** Числові значення, описи, підписи, введення даних
- **Стиль:** Чистий, нейтральний sans-serif
- **Ваги:** 300–700
- **Характеристики використання:**
  - `font-bold` для числових значень потужності (наприклад, "3.2 kW")
  - `font-medium` для вторинних підписів
  - `text-sm` як стандартний розмір тіла

## 4. UI Компоненти

### 4.1 GlassCard (Базовий контейнер)

**Файл:** `src/components/GlassCard.tsx`  
**Призначення:** Універсальний контейнер для будь-якого контенту

**Візуальні характеристики:**
- Фон: `hsl(220 20% 12% / 0.6)` — 60% непрозорість
- Розмиття: `backdrop-filter: blur(20px)`
- Межа: `1px solid hsl(220 15% 22% / 0.4)` — ледь помітна
- Закруглення: `rounded-xl` (12px)
- Внутрішній відступ: `p-5` (20px)

**Варіанти свічення (glowColor prop):**
- `solar` → `box-shadow: 0 0 20px hsl(45 100% 55% / 0.3), inset 0 1px 0 hsl(45 100% 55% / 0.1)`
- `grid` → `box-shadow: 0 0 20px hsl(200 100% 55% / 0.3), inset 0 1px 0 hsl(200 100% 55% / 0.1)`
- `battery` → `box-shadow: 0 0 20px hsl(155 80% 45% / 0.3), inset 0 1px 0 hsl(155 80% 45% / 0.1)`
- `load` → `box-shadow: 0 0 20px hsl(280 80% 60% / 0.3), inset 0 1px 0 hsl(280 80% 60% / 0.1)`

**Внутрішній inset glow** створює ефект "підсвічування зсередини верхньої кромки".

**Анімація входу:** Framer Motion — `opacity: 0→1`, `y: 10→0`, тривалість `0.4s`

```
Структура:
┌─────────────────────────────────────┐
│ ▔▔▔ inset glow (1px зверху)        │
│                                     │
│  [content]                          │
│                                     │
│  ░░░ blur(20px) крізь нього видно   │
│      елементи позаду                │
└─────────────────────────────────────┘
  ╰── outer glow (20px spread) ──╯
```

### 4.3 Energy Flow Diagram (Діаграма потоків)

**Файл:** `src/components/EnergyFlowDiagram.tsx`  
**Призначення:** Центральний візуальний елемент — відображає потоки енергії в реальному часі

#### 4.3.1 Вузли (Energy Nodes)

**Розташування (diamond pattern у SVG viewBox 0-100):**
- Solar (PV): `x=50, y=10` — зверху по центру
- Grid: `x=10, y=50` — зліва по центру
- Battery: `x=90, y=50` — справа по центру
- Load: `x=50, y=90` — знизу по центру

**Візуальне оформлення кожного вузла:**
- Контейнер: `.glass`, `rounded-2xl`, `p-4`, відповідний `.glow-*` клас
- Мінімальна ширина: `min-w-[110px]`
- Іконка: у колі `w-12 h-12`, фон `hsl(var(--color) / 0.15)`, іконка `w-6 h-6` кольором акценту
- Лейбл: `text-xs`, Orbitron, `tracking-wider`, колір акценту, `.text-glow-*`
- Значення: `text-lg`, `font-bold`, `text-foreground` — формат: `"3.2 kW"` або `"850 W"` (авто-конвертація при ≥1000W)

```
        ┌─────────────┐
        │   ┌─────┐   │
        │   │ ☀️  │   │  ← Кольорове коло з іконкою
        │   └─────┘   │
        │   SOLAR     │  ← Orbitron, glow
        │   3.2 kW    │  ← Inter bold
        └─────────────┘
           ╰ glow ╯
```

#### 4.3.2 SOC Індикатор

- Розташування: праворуч від вузла Battery (`left: 90%, transform: translate(30%, -50%)`)
- Контейнер: `.glass`, `rounded-xl`, `p-3`
- Лейбл: "SOC", `text-[10px]`, Orbitron, колір battery
- Значення: `text-2xl`, `font-bold`, `.text-glow-battery`
- Прогрес-бар: `w-12 h-1.5`, `rounded-full`, фон `bg-muted`, заповнення кольором `--battery`, анімована ширина

#### 4.3.3 Лінії потоків (SVG)

**Типи ліній:**
- Solar → Load (коли PV > 50W)
- Grid → Load (коли Grid > 50W, споживання з мережі)
- Solar → Grid (коли Grid < -50W, експорт у мережу)
- Solar → Battery (коли Battery > 50W, зарядка)
- Battery → Load (коли Battery < -50W, розрядка)

**Візуальне оформлення кожної лінії (2 шари):**

1. **Glow line (фоновий):**
   - `strokeWidth: 0.4`
   - `opacity: 0.15`
   - Колір: відповідний енергетичний
   - Створює ефект м'якого свічення

2. **Main line (основна):**
   - `strokeWidth: 0.2`
   - `strokeDasharray: "1.5 1"` — пунктирна
   - Заливка: `linearGradient` від 80% до 30% opacity
   - CSS фільтр: `drop-shadow(0 0 6px currentColor)` (клас `.neon-line`)
   - Анімація: `stroke-dashoffset` від 20 до 0, швидкість залежить від потужності

**Формула швидкості:**
```
speed = clamp(0.3, power / 2000, 2)
animation-duration = 1 / speed секунд
```
- 500W → speed=0.3 → повільний рух (3.3с)
- 2000W → speed=1 → середній (1с)
- 4000W+ → speed=2 → швидкий (0.5с)

#### 4.3.4 Частинки (Energy Particles)

**Для кожної лінії потоку генерується рухома частинка:**

- **Основна частинка:** `<circle r="0.8">`, opacity пульсує `0.9→0.4→0.9` за `0.5s`
- **Ореол:** `<circle r="1.5">`, `opacity: 0.15` — створює glow навколо
- **Рух:** `requestAnimationFrame` з лінійною інтерполяцією від точки A до точки B
- **Швидкість циклу:** `2000ms / speed` — синхронізовано зі швидкістю ліній

## 5. Анімації (детальний розбір)

### 5.1 Анімації компонентів (Framer Motion)

| Компонент | Trigger | Властивості | Тривалість | Easing |
|-----------|---------|-------------|-----------|--------|
| GlassCard | Mount | `opacity: 0→1, y: 10→0` | `0.4s` | default (ease) |
| Energy Node | Mount | `scale: 0→1` | spring | `type: "spring", delay: 0.1` |
| Tab content | Tab switch | `opacity: 0→1, x: 20→0` (вхід), `opacity: 1→0, x: 0→-20` (вихід) | `0.2s` | default |
| HEMS Mode Card | Hover | `scale: 1→1.02` | instant (spring) | — |
| HEMS Mode Card | Tap | `scale: 1→0.98` | instant (spring) | — |
| SOC Progress Bar | Value change | `width: old%→new%` | `1s` | default |

### 5.2 SVG Анімації (Energy Flow)

#### 5.2.1 Пунктирний рух ліній
```css
@keyframes energy-flow {
  0% { stroke-dashoffset: 20; }
  100% { stroke-dashoffset: 0; }
}
```
- **Ефект:** Пунктирна лінія "біжить" від джерела до споживача
- **Тривалість:** Динамічна, `1 / speed` секунд
- **Цикл:** `infinite`, `linear`

#### 5.2.2 Пульсація частинок
```xml
<animate attributeName="opacity" values="0.9;0.4;0.9" dur="0.5s" repeatCount="indefinite" />
```
- **Ефект:** Яскравість частинки плавно змінюється
- **Цикл:** Кожні 0.5 секунди

#### 5.2.3 Рух частинок (JavaScript)
- **Механізм:** `requestAnimationFrame` loop
- **Інтерполяція:** Лінійна між координатами (from.x→to.x, from.y→to.y)
- **Перезапуск:** Автоматичний при досягненні 100% (modulo duration)

### 5.3 CSS Анімації

#### 5.3.1 Повільний пульс (фоновий)
```css
@keyframes pulse-slow {
  0%, 100% { opacity: 0.6; }
  50% { opacity: 1; }
}
```
- **Використання:** Декоративні фонові елементи
- **Цикл:** Безкінечний, повільний

## 6. UX Патерни (детальний розбір)

### 6.1 Навігаційна модель

**Тип:** Single Page Application з табами  
**Парадигма:** Desktop-first з бічною навігацією

**Потік навігації:**
```
[NavigationRail] ──click──> [AnimatePresence] ──render──> [Tab Content]
                                    │
                          slide out old tab (-20px left)
                          slide in new tab (+20px right)
```

**Ієрархія вкладок:**
1. **Flow** (⚡) — Головний екран, енергетична діаграма. Перший і найважливіший.
2. **Dashboard** (📊) — Аналітика, графіки, KPI
3. **HEMS** (🧠) — Налаштування автоматизації

**UX рішення:**
- Навігація **завжди видима** (NavigationRail не ховається)
- Активна вкладка має **световий акцент** — користувач завжди знає, де він
- Переключення миттєве (200ms анімація) — відчуття responsive UI
- Немає вкладених навігацій чи модальних вікон — все плоско і прямо

### 6.2 Інформаційна архітектура

#### 6.2.1 Energy Flow — "Що відбувається зараз?"

**Ментальна модель:** Користувач бачить своїй будинок як енергетичну систему з 4-ма вузлами.

**Пріоритети сприйняття (зверху вниз):**
1. **Числа** — найбільші, `text-lg font-bold` — перше, що читає око
2. **Лінії руху** — анімація показує напрямок і інтенсивність потоку
3. **Кольори** — семантичне кодування дозволяє миттєво розпізнати тип
4. **Іконки** — підтвердження типу для нових користувачів

**Поріг активації ліній:** 50W — лінії не з'являються при мінімальних значеннях (шум, standby). Це запобігає візуальному хаосу.

**Динамічна швидкість:** Чим більше потужність → тим швидше рухаються частинки. Це дає **інтуїтивне відчуття масштабу** без читання цифр.

#### 6.2.2 Dashboard — "Як йдуть справи сьогодні?"

**Сканування (Z-pattern):**
```
[Solar] [Grid] [Battery] [Load]     ← Швидкий огляд живих значень
                                     
[Gen vs Consumption chart]  [Grid vs Battery chart]  ← Тренди
                                     
[Self%] [Autarky%] [Gen kWh] [Load kWh]  ← Підсумки дня
```

**UX рішення:**
- **Stat cards** вгорі — для швидкого scan (1-2 секунди)
- **Charts** у центрі — для аналізу трендів (10-30 секунд)
- **KPIs** внизу — для оцінки ефективності (5 секунд)

**Формат значень:**
- `< 1000W` → відображається як `"850 W"`
- `≥ 1000W` → конвертується в `"3.2 kW"` (один десятковий)
- Це запобігає когнітивному навантаженню від великих чисел

#### 6.2.3 HEMS — "Як налаштувати автоматику?"

**Потік взаємодії:**
```
1. Вибери режим → click на картку
2. Подивись/зміни налаштування → inputs з'являються контекстуально
```

**Контекстуальні налаштування (Progressive Disclosure):**
- **Adaptive:** Мінімум налаштувань (тільки minSoc) — "система сама розбереться"
- **Night Arbitrage:** + поля часу (offPeakStart, offPeakEnd) — "коли дешева електрика?"
- **Storm:** + stormReserve slider — "скільки тримати на чорний день?"

**UX рішення:**
- Картки режимів мають **тактильний відгук** (scale 0.98 при натисканні)
- Активний режим підсвічується **glow + ring** — неможливо переплутати
- Налаштування з'являються тільки **релевантні до обраного режиму** — немає disabled полів
- Range sliders показують **поточне значення у відсотках** поруч — не потрібно вгадувати

### 6.3 Realtime Data Flow

**Інтервал оновлення:** кожні 3 секунди (через `setInterval` у `InverterContext`)

**Що оновлюється:**
- PV Power: `1000–5000W` (випадкове, симуляція)
- Grid Power: `-1000–2000W` (негативне = експорт)
- Battery Power: `-1500–1500W` (негативне = розрядка)
- Load Power: `500–4000W`
- SOC: `20–95%`
- Voltages: `220–240V`
- Grid Frequency: `49.8–50.2 Hz`

**Візуальний ефект оновлень:**
- Числа змінюються миттєво (немає counting animation — це не маркетинг, а інструмент)
- Лінії потоків перебудовуються при зміні напрямку (наприклад, Grid: споживання → експорт)
- Графіки додають нову точку кожні 3 секунди (масив `history` росте)

### 6.4 Локалізація (i18n)

**Підтримувані мови:** English (EN), Українська (UK)

**Механізм:** Об'єкт `translations` у `InverterContext` з функцією `t(key)`.

**Перемикач:** Кнопка `Globe` внизу NavigationRail — один click тоглить мову.

**Локалізовані елементи:**
- Назви вузлів (Solar/Сонце, Grid/Мережа, Battery/Акумулятор, Load/Навантаження)
- Навігаційні мітки
- Назви режимів HEMS
- Мітки налаштувань
- Заголовки графіків
- KPI лейбли

### 6.5 Responsive поведінка

**Цільовий формат:** Desktop (Windows), resizable window

**Мінімальна підтримувана ширина:** ~800px (NavigationRail 80px + content)

**Масштабування:**
- NavigationRail: фіксована ширина 80px, завжди видима
- Energy Flow: SVG `preserveAspectRatio="xMidYMid meet"` — масштабується пропорційно
- Dashboard: `grid-cols-4` — колонки стискаються рівномірно
- Charts: `ResponsiveContainer width="100%"` — адаптуються до ширини
